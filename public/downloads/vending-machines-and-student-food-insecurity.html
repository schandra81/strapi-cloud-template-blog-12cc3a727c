Campus Food-Security Vending Implementation Pack

Model selection guide, planogram, FERPA checklist, and operator coordination workflow for student food-security vending

Universities are deploying vending as a complement to food pantries — anonymous, 24/7 access for students experiencing food insecurity, funded via meal-plan swipes, emergency-aid funds, or institutional health-services budgets. This pack bundles the four working documents (model selection, planogram, FERPA checklist, operator workflow) a campus food-security task force needs.

1. Funding-model selection guide

Four models are in common use at U.S. universities. Choose based on policy + funding source + privacy posture.

2. Eligibility + access workflow

  1. Eligibility determination

    Existing campus food-security task force or dean-of-students office determines eligibility; standard intake process; no academic-record tie-in.

  2. Voucher / credit issuance

    Eligible student receives QR code or campus-card credit; voucher capped at $X/day or $XX/week; refreshed each term.

  3. Cooler access

    Cooler accepts voucher as tender; transaction is anonymous from operator + university perspective (only voucher ID logged, not student identity).

  4. Privacy + dignity

    Cooler placement in low-stigma location (residence hall, library, student union — not inside food pantry only); SKU mix mirrors regular cooler to avoid 'free-food-only' stigma.

  5. Re-enrollment

    Eligibility refreshed each term; student case manager checks in on resource utilization; opt-in to additional supports (counseling, financial aid, basic-needs hub).

  6. Audit + accountability

    Aggregated utilization reported quarterly to task force; no student-identifiable data in reports beyond opt-in case management.

3. Food-security planogram (60 SKUs)

Planogram weights nutrient-density + shelf-stable + minimal-prep + cultural relevance. Allergen labeling per FALCPA; HHS-aligned where applicable.

Hydration — 18%
Bottled water (multi-size), electrolyte (low-cal), unsweetened sparkling. $1.00-$3.00.
Grab-and-go meal — 22%
Sandwich, sushi tray, salad bowl, hummus + veg cup, hard-boiled egg cup, charcuterie cup, lentil bowl. $4.50-$9.00.
Shelf-stable meal — 15%
Mac-and-cheese cup, ramen (low-sodium options), instant noodles, microwaveable bowl, single-serve soup. $2.00-$5.00.
Fresh fruit + dairy — 13%
Apple, banana, orange, Greek yogurt, cottage cheese, cheese stick, whole fruit. $1.00-$3.00.
Protein + bar — 12%
RXBAR, Quest, KIND, peanut-butter cracker pack, jerky, single-serve protein packet. $2.00-$4.00.
Whole-grain snack — 10%
Whole-grain crackers, popcorn, granola bar, rice cake, trail mix. $1.50-$3.50.
Culturally relevant — 10%
Curated SKU mix matching campus demographics (halal, kosher, plant-based, regional flavors); refreshed quarterly with student-affairs input. $2.00-$5.00.

4. FERPA + GLBA + privacy checklist

Food-security vending touches sensitive student data. Privacy office must review before launch.

  • Operator-signed FERPA DPA; covers any student-ID data that traverses operator systems via voucher SDK
  • GLBA acknowledgment if voucher funding ties to financial-aid balance
  • Data minimization: only voucher ID + transaction event traverses operator; no student name, ID, or eligibility status
  • Retention: voucher-redemption data ≤ 90 days at operator; aggregated/anonymized longer per privacy office
  • Re-identification ban: no joining cooler transaction data with financial-aid or academic records without IRB approval
  • Disclosure log: any non-routine disclosure logged; FERPA-compliant audit trail
  • Eligibility process: kept inside campus student-affairs (not visible to operator); operator sees only opaque voucher tender
  • Reporting: aggregated utilization only; no student-identifiable reports leave campus privacy boundary
  • Annual privacy-office review of voucher data flow

5. Operator coordination workflow

Operator role
Provides cooler + planogram + voucher-redemption integration; emits anonymous transaction events; cannot tie transactions to student identity.
Voucher issuance system
Owned by campus student-affairs / dean of students / financial-aid; operator integration is one-way (voucher → cooler), not bidirectional eligibility lookup.
Reconciliation
Monthly: operator submits voucher-redemption file to campus food-security task force; campus reimburses operator per agreed rate (typically operator cost + small handling fee, e.g., 5%).
Restock cadence
Per cooler traffic; typically 2-3x/week + telemetry-triggered top-offs. Cold-chain rotation per SKU shelf life.
Stockout protocol
Tier-1 SLA (≤ 24 hr resolve) for any stockout in food-security cooler; operator on-call route during high-demand weeks (move-in, finals, holidays).
Dietitian audit
Semi-annual planogram audit by campus dietitian; HHS-aligned share + cultural-relevance + allergen-labeling check; remediation within 30 days.
Joint review
Monthly first 6 months, then quarterly: food-security task force + operator + dietitian + privacy office.

6. Communications + dignity framework

Food-security vending succeeds when it's low-stigma + dignified. Communications avoid 'free-food-for-needy-students' framing and instead position cooler as a campus basic-needs resource.

Cooler placement
Same locations as regular coolers (union, library, residence halls); cabinet wrap matches regular cooler aesthetic; no visible 'food-pantry' marking.
Eligibility comms
Plain-language web page on basic-needs hub; reachable from dean of students + financial aid + student health; intake process designed for one-stop interaction.
On-cooler messaging
Standard cooler-face signage; voucher acceptance noted alongside other tender options; no stigmatizing call-out.
RA + student-affairs staff training
Confidentiality, dignity, referral process; how to mention the basic-needs hub without singling out a student.
Faculty + advisor talking points
How to refer a student to basic-needs hub; what the hub offers (vending vouchers + pantry + counseling + financial aid).
Annual reporting
Aggregated utilization to task force + university board; case-study format with privacy-protected narrative.

7. Operator selection criteria — food-security context

  • Higher-ed food-security reference accounts (or basic-needs vending placements)
  • FERPA DPA + GLBA acknowledgment + SOC 2 Type II on file
  • Voucher / tender integration capability (custom tender type beyond campus card)
  • Smart-Snacks / HHS / culturally-relevant SKU sourcing capability
  • Allergen labeling + cultural-attestation accuracy
  • Refund automation: QR + SMS refund flow ≤ 24 hr credit-back
  • Insurance + food product liability $1M + cyber $5M
  • Commission structure: cost-plus or low-commission given subsidy economics

8. Quarterly + annual review framework

  • Quarterly: aggregated voucher-redemption utilization by cooler placement
  • Quarterly: planogram refresh based on velocity + cultural-relevance input from student-affairs
  • Quarterly: dietitian audit + HHS-alignment + allergen-labeling check
  • Annually: privacy-office review of voucher data flow + DPA scope
  • Annually: food-security task force report to board (aggregated only)
  • Annually: operator contract review + cost-plus / commission re-anchor
  • Annually: basic-needs hub program evaluation + integration with vending
