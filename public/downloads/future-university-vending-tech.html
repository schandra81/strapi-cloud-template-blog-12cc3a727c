University Vending Technology Roadmap Pack

2. RFP template — required clauses

Use the clauses below in any vending RFP. Each maps to an operator capability the university should require in writing.

Commission + minimum guarantee
Operator submits commission tier table per volume bracket; minimum-guarantee floor per placement; quarterly true-up.
Uptime SLA
98%+ per machine per month; service credit ($X/breach hour) for breach. Telemetry-driven measurement, not self-report.
Refund SLA
≤ 24 hr credit-back to original tender; QR + SMS refund flow at every cooler; refund dashboard shared with campus.
Campus card SDK
Operator certifies SDK integration with [campus-card vendor]; FERPA DPA signed; test in dev environment before contract execution.
PCI + security
PCI-DSS 4.0 AOC + SAQ-B-IP; SOC 2 Type II in good standing; SIG/HECVAT response; pen-test report shareable under NDA.
Sustainability + STARS reporting
Operator commits to quarterly sustainability report aligned to AASHE STARS OP-8 + OP-9 sub-credits (local share, plant-forward share, packaging recyclability, ENERGY STAR equipment).
Dietitian + Smart-Snacks alignment
Operator commits to ≥ 50% HHS-aligned facings; dietitian audit semi-annually.
Accessibility
ADA Section 308 attestation per cooler; VPAT for digital interfaces; annual accessibility audit.
Insurance + indemnity
$2M GL, $1M umbrella, cyber-liability $5M, food product liability $1M; additional-insured to university.
Transition + termination
180-day operator transition obligation on termination; data export + retention obligations clear.

3. Integration checklist — campus systems

  • Campus card vendor (Atrium / Blackboard / CBORD / TouchNet) SDK certified
  • Campus IdP (Okta / Azure AD / Shibboleth) SSO for operator admin portal
  • Campus VLAN provisioned for cooler fleet; egress whitelist to operator API + payment processor
  • Auxiliary-services finance GL mapping for commission + sustainability subsidies
  • Dietitian + dining services planogram review workflow
  • Residence life + library admin escalation tree for incidents + refunds
  • Campus brand standards approval workflow for cabinet wrap + signage
  • Sustainability + STARS coordinator reporting cadence (quarterly + annual)
  • Campus police + EHS coordination MoU for incidents + restock route
  • Privacy office FERPA DPA signed + GLBA acknowledgment on file

5. Pilot-to-scale workflow

  1. Phase 0 — current-state audit

    Inventory existing placements, volume, contract terms, uptime, complaints. Establish baseline.

  2. Phase 1 — pilot RFP (5 placements)

    Issue RFP for AI cooler pilot at 5 high-traffic placements (union, library anchor, athletics, two residence halls). Award + install within 6 months.

  3. Phase 2 — pilot operations (6 months)

    Run pilot; collect uptime, refund, velocity, dietitian audit. Joint operator + campus quarterly review.

  4. Phase 3 — scale RFP

    Convert pilot learnings into final RFP for campus-wide program; structure as multi-year contract with annual scope adjustment.

  5. Phase 4 — campus-wide install

    Phased install over 12-18 months; legacy combo replaced or refurbished; new placements lit up.

  6. Phase 5 — optimization

    Year 2+ runs on annual review cadence; planogram + commission true-up annually; capital refresh on 5-7 year cycle.

6. Sustainability + AASHE STARS reporting

  • OP-8 Sustainable Dining: % HHS-aligned, % plant-forward, % organic, % third-party-certified
  • OP-8: local/regional share within 250 miles
  • OP-9 Sustainable Procurement: ENERGY STAR equipment %; refrigerant GWP < 700 where supplied
  • Packaging: % curbside-recyclable, compostable, or refillable
  • Workforce: route-staff living-wage attestation
  • Energy: kWh per cooler per year; trend year-over-year
  • Annual operator sustainability report to STARS coordinator

7. Risk + contingency planning

Operator default
Termination-for-cause clause + 180-day transition obligation; campus retains data export rights; second-place RFP respondent on standby.
Technology obsolescence
Cooler refresh on 5-7 year cycle; operator commits to upgrade path on contract renewal.
Campus card vendor change
Contract requires operator to re-integrate with new campus card vendor within 12 months at no cost beyond reasonable implementation fees.
Sustainability mandate change
Contract amendment process if AASHE STARS or campus sustainability standard changes mid-contract.
Data breach
Operator cyber-liability $5M; 24 hr notification SLA; campus privacy office leads coordinated response.
Volume shortfall
Minimum-guarantee clause; placement consolidation flexibility built into contract.

8. Annual review + scope-change process

  • Annual joint review meeting: auxiliary services + dietitian + IT + sustainability + privacy office
  • Prior-year performance: revenue, commission, uptime, refund, dietitian audit, STARS metrics
  • Scope-change requests: new placements, removed placements, planogram refresh, integration additions
  • Commission-tier review based on actual volume vs tier thresholds
  • Renewal decision: extend, re-RFP, or terminate; planning horizon ≥ 12 months before contract end
This pack is informational. Auxiliary services + procurement + dietitian + IT + privacy + sustainability office should jointly review operator capability claims and attestations before contract execution. Roadmap timing should align with capital-budget cycles + existing contract terms.
