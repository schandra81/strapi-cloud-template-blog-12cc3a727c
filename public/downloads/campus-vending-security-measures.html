Campus Vending Security Pack

Physical + AI cooler + cyber framework, operator + campus police MoU template, cybersecurity baseline checklist, student communications kit

Campus vending security spans three layers: physical (theft, vandalism, tampering), AI cooler operational (vision-system integrity, payment fraud), and cyber (PCI scope, FERPA exposure, network segmentation). This pack bundles the four working documents an auxiliary services + campus IT team needs to evaluate and govern a campus vending program.

1. Physical security framework

Physical security applies to traditional vending and AI coolers. Most loss is opportunistic; the controls below are inexpensive and high-leverage.

  • Bolt-to-floor or wall anchoring per OEM spec for any unit > 400 lbs gross weight; tamper-evident hardware.
  • Camera coverage: ≥ 1 PTZ or fixed camera with 1080p+, 24/7 recording, 30+ days retention, FOV covers the cooler face + clear floor space.
  • After-hours lighting: ≥ 10 fc at cooler face during all open hours; minimum 5 fc when building access-controlled.
  • Access-control logging: card-swipe or fob log at the building/zone level; data retained ≥ 90 days for incident reconstruction.
  • Cash management: cashless-preferred mode; if cash-accepting, max bill capacity $200, daily armored-car-or-operator pickup, GPS bag tracking.
  • Tamper sensors: door-open sensor, tilt sensor, internal cooling fault sensor; alerts sent to operator NOC within 60 seconds.
  • Glass + cabinet rating: tempered low-iron glass; cabinet steel gauge ≥ 18 ga; lock cylinder UL 437 or higher.

2. AI cooler operational security

Vision-system integrity
Operator should attest to per-SKU recognition accuracy ≥ 99%, with weekly model-drift report and quarterly recalibration. Mis-charge dispute window 24-48 hr.
Identity + authentication
Apple Pay/Google Pay tokenized, EMV contactless, optional campus-card SDK. No PAN at cooler. Failed-auth lockout after 3 attempts in 60 seconds.
Loss detection
Shrink alert if a tray loses > $X per session without a transaction; reconciliation between vision events + payment events at session close.
Anti-tailgate
Door closes within 4 seconds of last entry; cooler will not begin new session for 2 seconds after close to prevent piggyback theft.
Refund integrity
Refund requires student attestation + operator review; refund-rate threshold > 2% triggers automatic operator escalation + planogram audit.
Vision-data retention
Operator should attest to ≤ 30-day retention of un-anonymized clip data, with hash-only retention thereafter for shrink models.

3. Cybersecurity baseline checklist

Map operator controls to NIST CSF 2.0 and PCI-DSS 4.0. Campus IT should require attestation and evidence on contract execution and annually.

  • PCI-DSS 4.0 AOC and SAQ on file (typically SAQ-B-IP for tokenized cashless). PAN never traverses campus network.
  • Network segmentation: cooler on isolated VLAN with no lateral path to campus admin networks; egress whitelist to operator API + payment processor.
  • Cooler firmware: signed firmware updates, vendor SBOM available, CVE-tracking process with operator commitment to ≤ 30-day patch window for High/Critical.
  • Operator workforce: SOC 2 Type II or ISO 27001 in good standing; annual pen test report shareable under NDA.
  • MFA on all operator portal accounts that touch campus telemetry or payment data; SCIM or SAML SSO to campus IdP preferred.
  • Data classification: cooler emits transaction events only (timestamp, SKU, price, anonymized session ID); no student-ID or name fields traverse operator systems unless covered by signed DPA.
  • FERPA exposure assessment: where campus-card SDK integrates student ID, processed by FERPA-attesting operator with signed DPA; no profile re-identification.
  • Incident-response plan: operator notifies campus IT + auxiliary services within 24 hr of any security event affecting cooler data or PII.

4. Operator + campus police MoU — template clauses

MoU sets the protocols for incident response, evidence-handling, and recurring coordination. Campus police should sign before placement go-live.

Incident notification
Operator notifies campus police dispatch within 15 minutes of any tilt, forced-open, or vandalism alert. Police notifies operator dispatch within 15 minutes of any incident report involving cooler.
Evidence preservation
Operator preserves cooler-side video + transaction log ≥ 90 days for incidents under investigation; chain-of-custody form attached.
Camera access
Campus police can request relevant clips via written request to operator; turnaround ≤ 24 hr for active investigations.
Routine coordination
Quarterly meeting between operator account manager + campus police community liaison; review incidents, near-misses, recommended controls.
Restock + cash transport
Operator route schedules shared with campus police on a rolling 30-day basis; route changes notified ≥ 24 hr in advance.
Emergency lockout
Campus police can request remote-lock of a cooler during active incident; operator response time ≤ 15 min, 24/7.

5. Student communications kit

Cooler-face signage
11x17 inside-glass panel: 'How it works,' allergen note, refund QR, 24/7 support number, campus-police non-emergency line.
Refund flow card
4x6 lobby card at every cooler with QR to refund portal + SMS short-code.
Newsletter / orientation copy
150-word piece for residence-life newsletter explaining payment, refunds, allergen info, and what students should do if a cooler malfunctions.
Incident reporting card
Step-by-step: photo of cooler + slot, text to operator + non-emergency campus police if vandalism or unsafe condition.
Accessibility note
Students with mobility, visual, or hearing accommodations contact accessibility-services number printed on cooler face; alternative-format refund flow available.
Privacy note
Plain-language 200-word note explaining what data the cooler collects, retention, and the campus DPA terms; published on auxiliary-services site + QR-linked from cooler.

6. Quarterly security review checklist

  • Physical incident log (theft, vandalism, tampering) — count + dollar exposure + resolution
  • Cyber incident log (operator-side or cooler-side) — count + remediation
  • Refund rate by cooler; flags for > 2% threshold
  • Vision-system accuracy report + recalibration log
  • Firmware patch level + CVE coverage report
  • Camera + lighting + signage audit photos per placement
  • MoU action items + open items from campus-police meeting

7. Procurement attestation summary

  • Operator-signed PCI-DSS 4.0 AOC + SAQ
  • Operator-signed FERPA DPA + GLBA acknowledgment (financial-aid placements)
  • Operator SOC 2 Type II report under NDA
  • ADA Section 308 attestation per cooler
  • Insurance certificate: $2M GL, $1M umbrella, cyber-liability $5M, additional-insured to campus + auxiliary-services LLC
  • MoU executed by operator + campus police chief or designee
  • Quarterly review calendar booked for the contract term
This pack is informational. Campus IT, campus police, auxiliary services, and the institutional privacy office should jointly review operator attestations before contract execution. Operator capability claims should be verified through reference calls and through SOC 2/PCI evidence on file.
