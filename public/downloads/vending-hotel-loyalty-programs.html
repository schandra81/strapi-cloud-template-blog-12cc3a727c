Hotel Vending Loyalty Integration Playbook

Brand SSO matrix + accrual scope + tier benefit framework + folio wiring checklist

Loyalty integration turns a vending placement into a brand-engagement touchpoint. This playbook codifies how to wire vending into the property's loyalty program — single sign-on (SSO), points accrual, elite tier benefits, and folio post — across the five major brand systems. Each section maps to a capability your operator should be equipped to attest to.

1. Brand SSO + identity matrix

Loyalty integration starts with identity. The operator app or kiosk must recognize the guest's brand-loyalty number. Each brand has a different SSO posture.

2. Accrual scope decision

Brands differ on whether ancillary spend earns base or bonus points. Operator and property must agree on accrual scope before integration.

  1. Base-rate accrual on full ancillary spend

    Bonvoy: 10 points per USD on most ancillary spend at participating properties; Honors: 10 base points per USD; World of Hyatt: 5 base points per eligible USD. Confirm with brand contact at the franchise level.

  2. Elite multiplier eligibility

    Some brands apply elite tier multipliers (e.g., Bonvoy Platinum 50% bonus) to ancillary spend; others limit multipliers to room rate. Read the brand member terms before quoting.

  3. Promotional bonus campaigns

    Brand promotional campaigns (e.g., Bonvoy bonus stays) sometimes include ancillary; sometimes exclude. Operator dashboards should let property GM toggle inclusion.

  4. Non-participating purchase

    Sundries (toothpaste, charger cables) and alcohol are often excluded from accrual. Operator planogram tags non-accruing SKUs at the POS.

3. Elite tier benefit framework

Elite benefits at the vending touchpoint can be configured as one-time, daily, or per-stay. Capacity, brand standard, and folio impact must all be considered.

Welcome amenity (Gold / Diamond / Globalist tier)
One bottled water + one snack credit at check-in; PMS pushes credit to operator system; guest redeems at any pantry or vending touchpoint.
Daily complimentary water (most brand mid-tier+)
Operator system recognizes member ID at pantry checkout; 1 bottled water/day comped to folio with zero charge.
Free breakfast tier (varies)
Pantry grab-and-go counts toward breakfast benefit at some flags (Bonvoy Platinum at select brands); operator tags eligible SKUs.
Member-only pricing
10-15% discount on full pantry menu for members; non-members see full retail.
Birthday / anniversary recognition
Operator system reads brand-supplied lifecycle event and offers complimentary item.

4. Folio wiring checklist

  • Brand-certified middleware identified (Hapi, Infor Cloud, Stayntouch, Mews — depending on PMS)
  • PMS interface specification reviewed (Opera OXI / OnQ POSI / Concerto / ChoiceADVANTAGE)
  • Test environment staged with brand IT or franchise IT contact
  • Folio charge code mapped (typically 'sundry' or 'pantry' department; tax category set)
  • Accrual feed mapped (brand-loyalty backend post: number + transaction amount + property ID)
  • Comp redemption rules tested (zero-charge folio post with comp reason code)
  • Refund flow tested end-to-end (guest-app refund → folio reversal → loyalty point reversal)
  • Operator dashboard access for property GM + front-desk supervisor confirmed
  • Brand IT sign-off letter on file
  • Quarterly review cadence agreed with brand-standards liaison

5. Data + privacy framework

  1. Data minimization

    Operator receives loyalty number + transaction amount only — never the member's PII, payment card, or stay details. Brand backend handles identity resolution.

  2. PCI DSS

    Operator system PCI DSS Level 1 or 2 attestation on file. Card data tokenized; never traverses property network in clear text.

  3. State privacy laws (CCPA, CPRA, VCDPA, CPA, others)

    Operator + property privacy notices disclose vending data collection; consumer-rights request flow defined.

  4. Brand data-sharing agreement

    Brand reviews operator data-sharing posture before approving SSO. Annual recertification typical.
