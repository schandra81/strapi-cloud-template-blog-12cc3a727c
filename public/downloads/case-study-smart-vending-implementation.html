Smart Vending Implementation Pack

RFP template, SLA framework, QBR template, and hyper-care plan

Smart-vending implementations (telemetry + cashless + planogram analytics) succeed or fail on four documents: the RFP, the SLA, the QBR cadence, and the hyper-care plan. This pack codifies each. It maps to PCI DSS v4.0 (effective March 31, 2025), EMV + NFC, ADA Section 309.4, and NAMA-published service KPIs.

1. RFP scope statement

The scope statement is the single most-disputed RFP element. Specify hardware, telemetry, payment rails, integration, planogram method, and reporting in one paragraph each.

Hardware
Number of machines, form factor (glass-front, smart-cooler, micro-market kiosk, locker-pickup), age (≤ 5 years), ADA reach-range compliance attestation (Section 308).
Telemetry
Cellular-modem heartbeat ≤ 15 min, per-SKU velocity, online/offline status, payment-failure rate, temperature monitoring for refrigerated units.
Payment
EMV contact + contactless + mobile wallet + closed-loop. PCI DSS v4.0 AOC current within 12 months.
Integration
Identity (Okta / Azure AD / campus SSO), payroll deduct or pre-paid funding, HR/AP settlement cadence.
Planogram method
Data-driven quarterly refresh based on per-SKU velocity, healthy-share floor (NANA thresholds), allergen labeling per FALCPA.
Reporting
Operator dashboard + scheduled export to host BI (Tableau, Power BI, Looker). Schema published. Weekly delivery cadence.

3. SLA — measurable KPIs

  • Uptime ≥ 98% per machine, rolling 90-day window. Penalty: per-machine credit at 95-98%, equipment swap right below 95%.
  • Stockout incidents per machine per quarter — target ≤ 4. Measured by telemetry-flagged 'empty' selections.
  • Service-ticket SLA adherence ≥ 95% per quarter, measured per tier.
  • Refund rate ≤ 2% of transactions. Refund volume above 2% triggers planogram + machine quality review.
  • Cashless-rail uptime ≥ 99% (PCI-mandated systems). Outages > 1 hour reported within 24 hours.

4. Hyper-care plan (first 90 days post-go-live)

  1. Days 1-14 — daily on-site walkthrough

    Operator field rep on-site daily. Confirm telemetry connectivity, payment-rail function on every machine, signage placement, allergen labeling correctness. Member feedback intercepted in person at breakroom.

  2. Days 15-30 — telemetry baseline

    Per-SKU velocity baseline established. Stockout-pattern map drawn. First quarter-refresh candidates identified. Daily exception report to host PMO.

  3. Days 31-60 — first planogram refresh

    Bottom-10% velocity SKUs swapped for member-requested SKUs. Healthy-share floor maintained. A/B price-point test on 3-5 SKUs.

  4. Days 61-90 — QBR-1 + steady-state handoff

    First quarterly business review covers uptime, velocity, stockout, refund, SLA adherence. Hyper-care field rep transitions to regular route cadence.

5. Quarterly business review (QBR) template

  • Uptime per machine, per month, over the quarter (heatmap).
  • Velocity per SKU, per machine, with top-decile and bottom-decile flags.
  • Stockout incidents — count, machine, SKU, resolution time, root cause.
  • Refund volume — count, reason code, refund-rate by machine.
  • Service-ticket SLA adherence by tier (% met).
  • Payment-rail mix — cash, EMV, contactless, wallet, closed-loop — and trend.
  • Healthy-share consumption — % of transactions on healthy-flagged SKUs vs. planogram allocation.
  • Member feedback — NPS, refund-reason themes, request queue.
  • Planogram-refresh proposal for next quarter, with expected velocity lift.
  • Action items — owner, due date, success criterion.

6. Data + reporting requirements

Smart-vending value depends on host access to the data, not just the operator's dashboard. Specify schema, cadence, and ownership.

Schema
Transaction-level export: timestamp, machine ID, SKU, amount, payment rail, auth code, healthy-flag, refund flag. Documented schema delivered with first export.
Cadence
Weekly scheduled export to host SFTP or API endpoint by 09:00 Monday. Daily for high-volume sites.
Ownership
Host owns transaction data; operator licensed for service + de-identified benchmarking only. Data-retention period stated in DPA.
Member-data minimization
Operator stores employee ID + balance only on closed-loop; no SSN, no compensation, no benefit eligibility data.

7. Integration test plan (pre-go-live)

  • Identity SSO — 5 test members, each role tier, complete first transaction in under 6 seconds.
  • Payroll deduct / settlement — 1 full weekly cycle reconciled before go-live; AP confirms file format + matching.
  • Refund — member-initiated via app, SMS, and on-machine; each returns within 24 hours for cashless.
  • ADA — physical reach, operating force ≤ 5 lbf, audio-output verification on each kiosk.
  • Allergen labeling — SKU-level allergen flags display correctly on app + on-machine; FALCPA compliance attested.
  • Telemetry — heartbeat under 15 min; temperature alert tested via deliberate excursion; payment-failure alert tested.

8. Pre-signing red flags

  • Operator declines to provide PCI DSS v4.0 AOC dated within 12 months.
  • SLA proposed without measurable KPIs or remedies.
  • QBR cadence informal / verbal, no template, no action-item tracking.
  • Telemetry export limited to summary dashboard with no transaction-level data.
  • Hyper-care plan not in scope or limited to phone support.
