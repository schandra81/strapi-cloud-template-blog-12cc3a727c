Subscription Vending Gym Pack

Tier-platform integration + three subscription structures + abuse-prevention + monthly reconciliation

Subscription vending bolts a recurring entitlement (e.g., 1 shake per day, 10 bars per month, unlimited hydration) onto the gym membership. It lifts vending velocity 25-50% but introduces abuse-prevention, reconciliation, and platform-integration requirements. This pack covers the three structures, the platform stack, and how to keep the economics honest.

1. Three subscription structures — pick before signing

All subscription models share one challenge: gym pays the operator for product, gym charges member as subscription. The three structures differ in how that math runs.

StructureMechanicsGym riskOperator risk
A. Add-on entitlementMember pays $X/month bolt-on; entitled to N items/month at machineLow (member-paid)Reconciliation only
B. Bundled in tierPremium tier auto-includes vending allowance; gym absorbs costMedium (margin compression)Low
C. Pure pass-throughMember subscribes via gym app; gym remits per-redemption to operatorLowMedium (cash flow lag)

2. Tier-platform integration stack

Subscriptions only work if the gym software, the member-app, and the operator's vending platform exchange entitlement + redemption events.

  1. Gym software (entitlement source-of-truth)

    Mindbody Business, Glofox, ClubReady, GymMaster, MyZone, Wodify. Manages subscription billing + tier rules. API exposes 'member X has entitlement Y this month'.

  2. Member app (member-facing surface)

    App reads entitlement balance + shows remaining redemptions. Authenticates member at machine (fob, QR, deep-link).

  3. Operator vending platform (machine + redemption)

    Nayax, Cantaloupe, 365 Retail Markets, AWM Smart Shelf. Receives entitlement lookup at vend time; deducts allowance; posts redemption event.

  4. Reconciliation system

    Monthly join of gym billing data + operator redemption data; identifies orphan redemptions, missed entitlements, suspected abuse.

3. Abuse-prevention controls

An entitlement bolt-on without abuse controls bleeds margin within 30 days. Build these into the contract + the integration.

  • Daily cap: max N redemptions per member per 24-hour period; configurable per SKU (e.g., 1 shake/day, 2 bars/day).
  • SKU eligibility list: not every machine SKU is in the entitlement (e.g., premium recovery + ambient bars in; cold-brew coffee out).
  • Geofence: redemption only valid when member checks in (gym software check-in event within last 4 hours).
  • Rolling-30-day cap: prevents stockpiling at month-end then double-dipping early next month.
  • Velocity flag: any member exceeding 95th percentile redemption rate -> auto-review email + temporary hold pending review.
  • Member-tier validation at vend: real-time check vs gym software; lapsed members blocked.
  • Audit log: every entitlement check + redemption event logged with timestamp, machine ID, member ID for 24 months.

4. Pricing math — what the gym actually clears

5. Monthly reconciliation framework

Run reconciliation between days 5-10 of each month. Pull three data sources, join on member ID + month, identify variances.

  1. Pull gym billing report

    All subscriptions billed in the month, by member ID + plan.

  2. Pull operator redemption report

    All entitlement-redemptions in the month, by member ID + machine + SKU + timestamp.

  3. Pull gym software check-in events

    Member check-in events; validates redemption-vs-check-in correlation.

  4. Join + identify variances

    Orphan redemptions (redemption without active subscription), missed entitlements (subscription paid but zero redemptions), abuse flags (cap exceeded).

  5. Resolve + post adjustments

    Refund operator for redemptions out of scope; credit members for service outages; net commission line item updated.

  6. Quarterly trend review

    Plot redemption rate by member cohort, by month, by SKU; tune cap + SKU eligibility list.

6. Operator capabilities required

  • Real-time entitlement-lookup API at vend time (sub-second response or graceful timeout).
  • Daily + rolling-30-day cap engine — configurable per plan.
  • SKU eligibility list per plan — configurable.
  • Geofence / check-in window validation.
  • Member-ID authentication via fob, QR, or member-app deep-link.
  • Monthly redemption report in machine-readable format (CSV / Parquet) for the gym's reconciliation pipeline.
  • PCI-DSS attestation maintained; gym is not in PCI scope for the cashless portion that bypasses the subscription.
