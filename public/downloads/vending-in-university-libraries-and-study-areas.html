University Library AI Vending Pack

Placement criteria, equipment spec, finals-week SOP, campus card integration checklist, library administration approval framework

Library + study-area AI coolers face a unique set of constraints: silent operation, food-policy alignment, finals-week surge, and the library administration's approval process. This pack bundles placement criteria, equipment spec, finals-week operating SOP, campus-card integration checklist, and library administration approval framework.

1. Placement criteria

  • Foot traffic ≥ 500 unique students/day during academic term (gate count or building-access log)
  • Operating hours: cooler runs ≥ 18 hr/day during term; 24/7 during finals week if library extends hours
  • Location: lobby, atrium, or designated food-allowed zone — NOT inside silent study area or stacks
  • Acoustics: cooler ≤ 45 dBA at 3 ft (compressor + door + vision system combined); confirmed in equipment spec
  • ADA Section 308: 30x48 in clear floor space, operable parts 15-48 in AFF, ≤ 5 lbf operating force, accessible refund flow
  • Library food-policy alignment: confirm cooler placement does not contradict 'no food in library' policy (typical placements: lobby + designated cafe-adjacent zone)
  • Disposal: trash + recycling station within 25 ft of cooler; library facilities coordinates emptying cadence
  • Connectivity: cellular ≥ -95 dBm OR Ethernet drop on segmented VLAN; library Wi-Fi as backup

2. Equipment spec — library-optimized

Library AI coolers should be specified for low-noise operation, low-light visibility, and clean cabinet aesthetic.

Noise floor
≤ 45 dBA at 3 ft (compressor + door close + vision-system fan). Operator should attest to spec.
Compressor + refrigerant
Variable-speed compressor preferred; refrigerant GWP < 700 (R-290 or R-744 preferred); ENERGY STAR Tier-1 or comparable.
Lighting
Dimmable interior LED 4000K product-presentation; auto-dim 20% during overnight hours; no glare into adjacent study area.
Cabinet finish
Brushed stainless, matte black, or wood-grain laminate; anti-fingerprint coating; ADA-compliant operable parts.
Vision system
Edge inference; latency < 2 sec end-to-end; per-SKU accuracy ≥ 99%; mis-charge dispute window 24-48 hr.
Payment + identity
Apple Pay, Google Pay, contactless EMV, campus card SDK tap target; PCI-DSS 4.0 SAQ-B-IP.
Cabinet footprint
36-60 in wide; depth 30 in; fits typical library lobby footprint without disrupting traffic flow.

3. Finals-week operating SOP

Finals week drives 2-3x normal cooler volume and longer operating hours. Plan extra cadence + dedicated coverage.

  1. T-14 days: planogram refresh for finals

    Operator pulls 30-day velocity, weights toward caffeine + protein + study-fuel SKUs; healthy-share floor maintained; cabinet wrap signage refreshed.

  2. T-7 days: restock cadence doubled

    2x/week becomes daily during finals; telemetry-triggered top-offs same-day; on-call operator route during overnight hours.

  3. T-7 days: comms + signage

    '24/7 fuel for finals' signage at cooler + library lobby; refund + escalation QR card on every lobby table.

  4. Finals week: dedicated coverage

    Operator account manager on-call for the week; library admin notified of dispatch number for after-hours issues.

  5. Finals week: free-water program

    If campus funds, free water during peak study hours (typically 8 PM - 2 AM); QR redemption tracked.

  6. T+3 days post-finals: review

    Velocity + uptime + refund rate reviewed; planogram refreshed for break period; capital + equipment audit.

5. Campus-card integration checklist

  • Campus card vendor (Atrium / Blackboard / CBORD / TouchNet) SDK certified
  • Tender support: dining dollars, declining balance, flex; meal exchange if applicable
  • Library-specific funding line (if applicable): library dean or student-affairs sub-account for free-water / finals-week programs
  • FERPA DPA signed by operator; covers student-ID data via campus card SDK
  • Settlement: daily file to campus card vendor; library admin receives monthly summary if separate funding line
  • Apple Pay + Google Pay alongside campus card target
  • Refund flow: QR + SMS refund flow to original tender; ≤ 24 hr credit-back; library admin sees refund dashboard for cooler placements in library
  • Loyalty (optional): healthy-share double-points during finals; library partnership messaging

6. Library administration approval framework

Library administration approves cooler placement on alignment with food policy, acoustics, aesthetics, and student-life calendar. Use the framework below to navigate approval.

Initial pitch
Auxiliary services + operator present placement plan to library dean + dean of student services; covers volume forecast, food-policy alignment, noise spec, capital + commission structure.
Food-policy alignment
Cooler placement in 'food-allowed' zones (lobby, atrium, cafe-adjacent); operator confirms allergen labeling + cabinet cleanliness cadence.
Acoustics + aesthetics
Equipment spec sheet attached; ≤ 45 dBA at 3 ft attested; cabinet wrap submitted to library admin + campus brand standards for approval.
Trash + recycling
Library facilities confirms trash + recycling station placement; operator agrees to twice-weekly trash audit during high-traffic periods.
Pilot proposal
Single-cooler pilot for 90 days before scaling; KPI scorecard (uptime, refund, food-policy incidents, complaints); pilot review with library admin at day 30 + day 90.
Communications + signage
Cooler-face signage clearly states food-allowed zones; library admin reviews + approves before launch.
Joint review cadence
Monthly first 90 days; quarterly thereafter. Library admin + auxiliary services + operator account manager.

7. Communications + signage kit

  • Cooler-face panel: payment options, refund QR, allergen summary, library food-allowed zone reminder
  • Library lobby signage: 'fuel for studying — open until [hours]'
  • Finals-week refresh signage: 'open 24/7 during finals'
  • Food-allowed zone signage: arrows + map showing where food may be consumed in library
  • Refund QR card at lobby table + circulation desk
  • RA / library-staff training packet (30 min) on refund + escalation tree
  • Newsletter feature in library + residence-life newsletter at term start + finals

8. Quarterly review + KPI scorecard

  • Uptime per cooler per month
  • Transactions per day + basket size trend
  • Top-15 / bottom-15 SKU velocity
  • Refund rate (target < 2%)
  • Food-policy incidents (target zero)
  • Library complaint volume (target zero)
  • Finals-week surge performance (transactions vs prior term)
  • Cabinet cleanliness audit (operator + library facilities walk every quarter)
  • Brand-standard compliance (cabinet wrap condition)
